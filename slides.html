1–3
Stages 1 – 3
Ingestion
Turns any supported content format into a clean, structured, English-language body ready for fact-checking — regardless of source format, file type, or language.
Pipeline
1
Accept input URL, file upload, or plain text. Raw content is archived to S3.
URLPDF DOCXHTML MDRTF ImageText
2
Extract & structure Parse text from the file format. Claude extracts metadata (title, publisher, author, date, section, opinion flag) and detects the source language.
3
Summarise if needed If the article body exceeds ~4 000 tokens, Claude summarises it so downstream agent prompts stay within token limits. Shorter articles pass through unchanged.
Distinctive features
Claude OCR for image inputs
Articles submitted as screenshots or photos are read by Claude's vision API, extracting every visible word verbatim. No dedicated OCR library required — accuracy matches the LLM's own reading comprehension.
Automatic multilingual translation
Language is detected from the extracted text, not from HTTP headers or metadata. Non-English articles are translated to English by Claude before any analysis runs. The original-language text is preserved separately — investigators see clean English while the archive retains the source.
Article body preserved for writing quality
The full English body is stored on content.body and passed to the Judgement stage, where Claude independently assesses writing style signals (sensationalism, anonymous sources, emotional language) as supplementary credibility context — separate from the factual claim analysis.
4
Stage 4
Investigation
Four specialised agents run in parallel to find and classify evidence for every claim — prioritising original data over second-hand mentions, and catching AI-generated content before it reaches the scorer.
Parallel agents
Search agent Tavily search per claim; chases citations to primary sources; extracts full text from HTML, PDF, CSV, and Excel URLs.
Fakeness agent GPTZero API scores the article's likelihood of being AI-generated (0–100). Scores above 60 trigger a credibility penalty in Judgement.
Statistics agent Extracts cited figures and checks them against authoritative data sources to flag numbers that appear inaccurate or out of context.
Source checker Researches the publisher's and author's track record. Produces a 0–100 credibility score, political bias label, and known-issue list.
Citation-chasing evidence tiers
Primary — kept as strong evidence
Source provides its own original data, analysis, or official statement. Weight multiplier: 1.2×
Secondary with citation — followed
Source relays a claim and links to a specific URL. The agent follows the link and reclassifies the target — up to MAX_HOP_DEPTH = 2 hops. Weight multiplier: 0.8×, with an additional 0.9× decay per hop.
Mention-only — discarded
Source references a claim without any citation trail. Carrying no traceable evidence, it is dropped entirely rather than contributing noise.
Distinctive features
Binary file extraction
PDF, CSV, and Excel URLs are detected by path extension and downloaded via httpx, then parsed with pypdf / stdlib csv / openpyxl. Statistical reports and government data sheets are readable as evidence — not just HTML pages.
Deduplication + retry
Sources are fingerprinted by the first 300 characters of extracted text to catch syndicated copies. If unique sources fall below MIN_SOURCES_PER_CLAIM, Claude generates a rephrased query and retries Tavily up to MAX_SEARCH_RETRIES times — maximising evidence coverage per claim.
Source independence tracking
Each source is flagged is_independent when it is editorially independent of the original publisher. Non-independent corroboration (e.g. the same outlet's sister publication) is surfaced in the evidence trail, preventing self-referential validation from inflating the score.
5
Stage 5
Judgement
Aggregates all evidence into a weighted, multi-factor credibility score — distinguishing between claims with conflicting evidence and claims with no evidence, and surfacing structural signals that a simple score cannot capture.
Scoring chain (per claim)
1
Evidence extraction Claude selects the most relevant verbatim snippet from each source and determines whether it supports or contradicts the claim.
2
Weighted source scoring Each relevant source contributes a signed value:
base_weight × direction × quality_multiplier × hop_decay
3
Net confidence multiplier Adjusts the normalised score by how many independent sources agree vs. disagree — from 0.6× (contradicting majority) to 1.4× (5+ corroborating).
4
Final blend Claim scores are averaged, then blended with publisher credibility and penalised if AI-generated content is detected.
claims × 0.7 + publisher × 0.3 [× 0.8 if fakeness_score > 60]
Distinctive features
Inconclusive vs. Unverified — semantically distinct
Most systems collapse these into one label. FactGuard separates them: Inconclusive means sources were found but their evidence is balanced or conflicting (score 45–59). Unverified means no usable evidence was found at all (max_possible_sum = 0, score forced to neutral 50). The reader knows whether the system looked and was uncertain, or simply could not look.
Government-source-only flag + boost
When all corroborating sources for a claim are government-owned, a 1.1× credibility boost is applied (official sources carry authority), but government_source_only_flag is also surfaced in the UI — prompting the reader to seek independent verification where official sources may have a conflict of interest.
Conflict detection
When the evidence-based claim score and the publisher credibility score diverge by 40+ points, is_conflicted = true is set and a note is written into the summary. This catches the two hardest cases: a known-bad outlet publishing genuinely accurate claims, and a credible outlet running a poorly-evidenced story.
Claim verdict scale
True  80–100
Mostly True  60–79
Misleading  30–44
Inconclusive  45–59
Unverified  no evidence
Mostly False  15–29
False  0–14
