Stages 1 – 3
Ingestion
Turns any supported content format into a clean, structured, English-language body ready for fact-checking — regardless of source format, file type, or language.
Pipeline
1
Accept input URL, file upload, or plain text. Raw content is archived to S3.
URLPDF DOCXHTML MDRTF ImageText
2
Extract & structure Parse text from the file format. Claude extracts metadata (title, publisher, author, date, section, opinion flag) and detects the source language.
3
Summarise if needed If the article body exceeds ~4 000 tokens, Claude summarises it so downstream agent prompts stay within token limits. Shorter articles pass through unchanged.
Distinctive features
Claude OCR for image inputs
Articles submitted as screenshots or photos are read by Claude's vision API, extracting every visible word verbatim. No dedicated OCR library required — accuracy matches the LLM's own reading comprehension.
Automatic multilingual translation
Language is detected from the extracted text, not from HTTP headers or metadata. Non-English articles are translated to English by Claude before any analysis runs. The original-language text is preserved separately — investigators see clean English while the archive retains the source.
Article body preserved for writing quality
The full English body is passed to Judgement, where Claude independently assesses writing style signals (sensationalism, anonymous sources, emotional language) as supplementary credibility context — separate from the factual claim analysis.
4
Stage 4
Investigation
Four specialised agents run in parallel to find and classify evidence for every claim — prioritising original data over second-hand mentions, and catching AI-generated content before it reaches the scorer.
Parallel agents
Search agent Tavily search per claim; chases citations to primary sources; extracts full text from HTML, PDF, CSV, and Excel URLs.
Fakeness agent GPTZero API scores the article's likelihood of being AI-generated (0–100). Scores above 60 trigger a credibility penalty in Judgement.
Statistics agent Extracts cited figures and checks them against authoritative data sources to flag numbers that are inaccurate or out of context.
Source checker Researches the publisher's track record. Produces a 0–100 credibility score, political bias label, and known-issue list.
Distinctive features
Citation chasing to primary sources
Each Tavily result is classified into a tier. Primary sources (own data or official statement, 1.2× weight) are kept. Secondary sources that cite a URL are followed up to 2 hops (0.8× × 0.9 per hop). Mention-only sources with no citation trail are discarded — preventing noise from inflating the score.
Binary file extraction
PDF, CSV, and Excel source URLs are downloaded directly via httpx and parsed before classification. Statistical reports and government data sheets become readable evidence — not just HTML pages.
Deduplication + smart retry
Sources are fingerprinted to discard syndicated copies. If unique sources fall below the minimum, Claude generates a rephrased query and retries Tavily — maximising independent evidence coverage per claim.
5
Stage 5
Judgement
Aggregates all evidence into a weighted, multi-factor credibility score — distinguishing between claims with conflicting evidence and claims with no evidence, and surfacing structural signals that a simple score cannot capture.
Scoring chain
1
Evidence extraction Claude selects the most relevant verbatim snippet from each source and determines whether it supports or contradicts the claim.
2
Weighted source scoring Each relevant source contributes a signed, quality-adjusted value. The net count of supporting vs. contradicting sources applies a confidence multiplier (0.6× to 1.4×).
base_weight × direction × quality × hop_decay × net_multiplier
3
Final blend Claim scores are averaged, blended with publisher credibility (70/30), and penalised if AI-generated content is detected.
claims × 0.7 + publisher × 0.3 [× 0.8 if fakeness_score > 60]
Distinctive features
Inconclusive vs. Unverified — semantically distinct
Most systems collapse these into one label. FactGuard separates them: Inconclusive — sources found but evidence is balanced or conflicting (score 45–59). Unverified — no usable evidence found at all (score forced to neutral 50). The reader knows whether the system looked and was uncertain, or simply could not look.
Government-source-only flag + boost
When all corroborating sources for a claim are government-owned, a 1.1× credibility boost is applied — but government_source_only_flag is also surfaced in the UI, prompting the reader to seek independent verification where official sources may have a conflict of interest.
Conflict detection
When claim evidence score and publisher credibility score diverge by 40+ points, is_conflicted = true is flagged. This catches a known-bad outlet publishing accurate claims, or a credible outlet running a poorly-evidenced story.
